# 开发指南

如何搭建、开发和部署本站。

## 技术栈

- **框架**：Next.js 14.2（App Router）
- **语言**：TypeScript
- **数据库**：SQLite（通过 `better-sqlite3`）
- **认证**：NextAuth.js（credentials provider）
- **样式**：Tailwind CSS + `@tailwindcss/typography`
- **Markdown**：`react-markdown` + `gray-matter`（frontmatter 解析）
- **编辑器**：`@toast-ui/editor`
- **测试**：Vitest（node 环境）
- **部署**：Docker + Cloudflared tunnel（NAS 场景）

## 初始化

1. 克隆仓库
2. 安装依赖：
   ```bash
   npm install
   ```
3. 复制环境变量模板并填值：
   ```bash
   cp .env.example .env.local
   ```
4. 必填变量：
   - `ADMIN_PASSWORD` — 登录密码；请使用强密码（生产环境禁止 `changeme`）
   - `NEXTAUTH_SECRET` — 用 `openssl rand -base64 32` 生成
   - `NEXTAUTH_URL` — 如 `http://localhost:3000`
5. 可选但推荐：
   - AI 服务商变量（也可通过管理界面设置）
   - 发布到 X 所需：`X_CONSUMER_KEY`、`X_CONSUMER_SECRET`、`X_ACCESS_TOKEN`、`X_ACCESS_TOKEN_SECRET`

6. 启动开发服务器：
   ```bash
   npm run dev
   ```

## 数据库

SQLite schema 内联定义在 `lib/db.ts` 中。首次运行时若表不存在会自动创建。

表清单：
- `todos`、`files`、`albums`、`diary`、`fortune_history`
- `ai_providers`、`ai_chat_history`
- `subscription_sources`、`subscription_briefs`

`lib/db.ts` 通过 `stmts` 导出 prepared statements 以提升性能。

可用 `sqlite3 data/site.db` 直接查看数据库。

## 添加新功能

### 1. 新增 API 路由

创建 `app/api/<name>/route.ts`：

```typescript
export const runtime = 'nodejs';
import { getServerSession } from 'next-auth';
import { authOptions } from '@/lib/auth';
import db from '@/lib/db';
import { rateLimitByIp } from '@/lib/rateLimit';

export async function POST(req: Request) {
  const session = await getServerSession(authOptions);
  if (!session) return Response.json({ error: 'Unauthorized' }, { status: 401 });
  const rl = rateLimitByIp(req, 'my-feature', 20);
  if (rl) return rl;

  let body: any;
  try { body = await req.json(); } catch {
    return Response.json({ error: 'Invalid JSON' }, { status: 400 });
  }

  // ... 业务逻辑
  return Response.json({ ok: true });
}
```

### 2. 新增管理页面

创建 `app/admin/<feature>/page.tsx`：

```tsx
'use client';
import { useState, useEffect } from 'react';

export default function MyFeaturePage() {
  // ... UI
}
```

在 `app/admin/layout.tsx` 中添加导航链接。

### 3. 编写测试

创建 `tests/api/<name>/route.test.ts`：

```typescript
import { describe, it, expect, vi, beforeEach } from 'vitest';
import { mockSession, mockDbStmt } from '../../helpers';

describe('POST /api/my-feature', () => {
  it('未登录返回 401', async () => {
    mockSession(false);
    const { POST } = await import('@/app/api/my-feature/route');
    const res = await POST(new Request('http://localhost', { method: 'POST', body: '{}' }));
    expect(res.status).toBe(401);
  });
});
```

运行：`npm test`

## 质量门禁

根据改动规模运行对应检查：

```bash
npm run lint          # 格式、架构和 TypeScript 检查
npm run verify        # lint + Vitest + 生产构建
npm run verify:large  # verify + 完整 Playwright e2e
```

`npm run lint:architecture` 会执行可运行的架构边界检查，包括：

- 使用 SQLite、文件系统或流式响应的 API route 必须声明 `runtime = 'nodejs'`
- middleware 必须保持 Edge-safe，不能引入 Node-only 模块
- UI 和共享组件不能直接访问数据库层
- 样式保持 Tailwind-first，不引入 CSS modules
- 运行时技能以 `.codex/skills/` 为唯一来源

需要专门做架构/风格审查或上下文清理时，使用 `.codex/agents/architecture-reviewer.md`。

## 添加 AI 技能

1. 创建 `.codex/skills/<skill-name>/SKILL.md`：

```yaml
---
name: my-skill
description: 触发时机（必须具体——影响 AI 路由）
user_invocable: true
system: "System prompt"
prompt: "带 {{content}} 占位符的用户 prompt"
output: content
---

# My Skill

人类可读的技能说明。
```

2. 可选：在 `scripts/` 中绑定脚本，在 `references/` 中提供参考文档
3. 从代码使用：

```typescript
import { getSkill } from '@/lib/skills';
const skill = getSkill('my-skill');
// skill.system、skill.prompt、skill.output
```

## 运维工作流

- 只要改动影响功能、运维、测试或部署流程，就要在同一组变更中同步更新 README / 文档。
- 已完成的改动不应只停留在本地工作区，应提交并推送到 Git。
- `./deploy-to-nas.sh` 只用于大改动、发布级变更或明确要求的 NAS 部署，小改动通常到 Git 推送为止。
- 长时间运行、可能占用端口或遗留子进程的本地测试流程，应使用受控执行入口：

```bash
npm run dev:managed
npm run test:managed
node scripts/run-managed-command.mjs --label e2e-local --clear-port 3000 -- <你的 e2e 命令>
```

- 本地受控执行日志写入 `log/automation/`，NAS 部署日志写入 `log/deploy/`。
- 测试、e2e 和 NAS 部署结束后，要确认相关进程、占用端口、SSH / SFTP 会话以及临时目录都已彻底清理。

## 测试

```bash
npm test                 # 跑一次全部测试
npm run test:managed     # 带时间戳日志的测试入口
npm run test:watch       # watch 模式
```

测试 setup（`tests/setup.ts`）mock 了：
- `better-sqlite3`（所有 `db.prepare().*` 返回 vitest mocks）
- `next-auth`（`getServerSession`）
- `@/lib/rateLimit`、`@/lib/fetchers`、`@/lib/xapi`

`tests/helpers.ts` 提供的 helper：
- `mockSession(true|false)` — 设置认证状态
- `mockDbStmt({get, all, run})` — 设置 DB 返回值
- `mockRateLimit429()` — 强制触发频率限制
- `postReq(body)` — 构造 POST Request
- `mockStreamResponse(fetch)` — mock SSE 流式响应

## 部署

### Docker（本地 / NAS）

```bash
docker-compose up -d
```

参见 `Dockerfile` 和 `docker-compose.yml`。

### NAS + Cloudflared tunnel

```bash
./deploy-to-nas.sh
```

部署所需变量统一放在根目录 `.env.local`：
- `NAS_HOST`
- `NAS_USER`
- `NAS_PATH`
- `NAS_PASSWORD`
- `CLOUDFLARE_TUNNEL_TOKEN`

脚本会上传 `.env.local` 和 `docker-compose.nas.yml`，在 NAS 上构建 `my-site:latest`，然后执行 `docker compose --env-file .env.local -f docker-compose.nas.yml up -d`。
脚本还会把部署日志写入 `log/deploy/`，并在退出前清理远端暂存目录、关闭 SSH / SFTP 会话。

## 编码规范

- TypeScript 严格模式
- 用户不要求时不用 emoji
- 保持 bundle 小——优先原生 API 而非库
- 使用 Node.js API（db、fs）的路由必须声明 `export const runtime = 'nodejs'`
- 样式一律用 Tailwind——不使用 CSS modules
- 新增 API 路由必须带测试（认证检查 + 正常路径 + 错误场景）

## 常见坑

- `better-sqlite3` 是原生模块——只能在 Node runtime 使用，不能在 Edge middleware 中使用
- `useLocale` hook 读 cookie——依赖 locale 的 UI 需用 `suppressHydrationWarning` 避免 SSR 不匹配
- 使用流式响应（`text/event-stream`）的 API 路由需要 `runtime = 'nodejs'` 且正确处理 ReadableStream
- `.codex/skills/` 目录同时被 web 应用（`lib/skills.ts`）和 Codex 技能发现使用——技能需兼容两者
