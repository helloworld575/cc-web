## 公共功能

### 主题

- 使用桌面主题按钮或移动端导航抽屉，在浅色和深色模式之间切换。
- 首次访问遵循操作系统偏好；主动选择会保存到 `localStorage`，刷新后继续生效。

### 博客

- 访问 `/blog` 浏览全部文章
- 通过搜索框按标题或摘要搜索
- 默认按发布时间从新到旧排列，可切换为从旧到新
- 搜索和排序后按每页 10 篇分页
- 点击文章标题阅读全文
- 发布时间随中英文界面格式化，不受服务器时区影响
- 正文采用与编辑器兼容的 GFM 展示，支持表格、任务列表、删除线和自动链接

### 文件（图片）

- 访问 `/files` 查看公开图片
- 按相册分类组织
- 图片通过适合 CDN 缓存的 `/uploads/<filename>` 路径提供，并带有长期缓存

### 工具（`/tools`）

登录后可见五个 tab：

- **待办** — 任务列表，支持"已完成/未完成"筛选
- **日记** — 按日期记录，支持 markdown
- **命理** — 中国传统命理工具（八字、紫微、六爻、梅花易数）
- **AI 对话** — 与已配置的 AI 服务商聊天，并可重新打开已保存的历史记录
- **订阅** — AI 生成的订阅源摘要

## 管理后台

登录密码后访问 `/admin/*` 进入管理面板。

| 页面 | 用途 |
|------|------|
| `/admin/blog` | 创建、编辑、删除博客文章（带 markdown 编辑器）|
| `/admin/diary` | 管理日记 |
| `/admin/tools` | 管理待办列表 |
| `/admin/files` | 上传图片，组织相册 |
| `/admin/skills` | 编辑 AI 技能（system prompt、模板）|
| `/admin/ai-config` | 配置 AI 服务商（OpenAI、Anthropic）|
| `/admin/subscriptions` | 添加、编辑、删除订阅源 |
| `/admin/x-post` | 通过 AI 生成推文并发布到 X/Twitter |

## AI 对话

1. 前往 **管理后台 → AI 配置**
2. 点击 **+ 新增服务商**
3. 可以先点击 Right Code GPT-5.5 或 Claude 预设，也可以手动填写：
   - **名称** — 如"Claude"
   - **API 类型** — `openai` 或 `anthropic`
   - **API URL** — 服务基础 URL（如 `https://www.rightapi.ai/codex` 或 `https://www.rightapi.ai/claude`）
   - **API Key** — 密钥
   - **模型** — 如 `gpt-5.5` 或 `claude-opus-4-8`
4. 至少将一个服务商勾选为 **默认**
5. 点击 **测试连接** 验证
6. 点击 **保存**

然后前往 **工具 → AI 对话**，从下拉框选择服务商开始聊天。如果 `.env.local` 配置了 `CLAUDE_API_KEY` 或 `RIGHT_CODE_GPT_API_KEY`，env provider 会排在前面，并由第一个 env provider 保持为默认服务商，同时数据库中保存的其他 provider 仍可选择。完成的对话会自动保存，按服务商分组，可从历史记录列表重新打开或删除。聊天面板会固定输入区，消息时间线可独立滚动，Markdown 表格以及 fenced CSV/TSV 输出会渲染为更易读的表格。历史记录保留完整转录，但发送给上游模型时只携带最近的对话窗口，以降低上下文占用。

## 订阅系统

追踪博客、GitHub 仓库、X 账号等源的最新动态。

### 添加订阅源

1. 前往 **管理后台 → 订阅**
2. 点击 **+ 新增源**
3. 填写：
   - **名称** — 显示名
   - **URL** — 源地址
   - **订阅主题** — `AI 订阅` 或 `安全订阅`
   - **分类** — `rss`、`github`、`selfblog`、`newsletter`、`reddit` 等抓取类型
   - **抓取间隔** — 源元数据；每日发布按配置的上海时区计划执行
4. 点击 **保存**

### 查看摘要

1. 前往 **工具 → 订阅**
2. 点击 **全部刷新** 抓取并生成最新摘要
3. AI 生成的摘要按订阅主题自动选择模板：
   - **安全订阅** — 漏洞通告、威胁情报、安全事件、防御研究
   - **AI 订阅** — 模型与产品、研究与评测、开源工程、行业与治理

`subscription` 是不可直接调用的路由 Skill；实际摘要分别由 `subscription-security` 和 `subscription-ai` 生成。混合批次会先验证所有需要的 Skill，再调用默认 AI 服务商，避免只生成或写入半个批次。

每日定时发布不依赖手动摘要：所有源抓取完成后，系统会各生成一篇 AI 日报和安全日报。不同类别展示各自的事实字段，并使用类别配额和分类内来源轮询，防止一种内容或单个高频来源占满日报。正文保留明确、可点击的参考链接，只写来源可核实的事实；编辑判断只允许出现在片头。

## 发布到 X

将博客或日记转为推文并发布。

1. 前往 **管理后台 → Post to X**
2. 选择来源：**博客**、**日记** 或 **自定义**
3. 选中一条（或粘贴自定义内容）
4. 点击 **🤖 用 AI 生成推文** — 生成单条推文或推文串
5. 根据需要编辑（字数会实时显示）
6. 附加图片：
   - **📁 从本地电脑** — 选择本地文件
   - **🖼️ 从站点文件** — 从已上传的图片中选
7. 点击 **🐦 发布到 X**

### 前置条件

- `.env.local` 中配置 `X_CONSUMER_KEY`、`X_CONSUMER_SECRET`、`X_ACCESS_TOKEN`、`X_ACCESS_TOKEN_SECRET`
- X App 权限必须为 **Read and Write**

## 命理工具

**工具 → 命理** 包含四种占卜方式：

- **八字** — 根据生辰进行四柱分析
- **紫微斗数** — 紫微星盘
- **周易六爻** — 卦象占卜
- **梅花易数** — 基于数字的快速占卜

每种方式使用专门的 AI 技能进行详细解读。
